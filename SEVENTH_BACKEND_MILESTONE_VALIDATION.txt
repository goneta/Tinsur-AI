Tinsur.AI Seventh Backend Milestone Validation
=============================================

Scope
-----
Milestone 7 implements automatic policy document and certificate packet orchestration after product-catalog policy acquisition. The acquisition flow now supports generating policy documents immediately after an approved catalog quote issues a policy, while returning existing policy documents idempotently on repeated acquisition requests.

Implemented Capability
----------------------
1. Product policy acquisition requests now include `generate_policy_documents` and `regenerate_policy_documents` controls.
2. Product policy acquisition responses now include `document_status` and `documents` metadata so callers can retrieve policy certificate details from the same acquisition response.
3. A new `ProductPolicyDocumentPacketService` reuses the existing template-backed document generator, verifies tenant scope before document generation, returns existing policy documents without duplicate generation, and serializes compact document metadata.
4. The acquisition service invokes the document packet service only when a policy exists and document generation is requested, preserving quote-only behavior for manual or referred acquisitions.

Focused Test Coverage
---------------------
- Request defaults enable document generation but do not regenerate documents by default.
- Existing policy documents are returned idempotently without calling the generator.
- Missing policy documents are generated and surfaced through response-safe metadata.
- Cross-tenant policy document packet generation is rejected.
- Acquisition responses include generated document packet metadata.
- Acquisition orchestration can skip document generation or report documents unavailable until a policy exists.

Remaining Risks
---------------
The document generator depends on configured policy document templates. If a tenant has no active templates, the orchestration returns `no_templates` with no document metadata rather than fabricating policy documents. This is intentionally conservative and preserves the existing template-backed document-generation model.
