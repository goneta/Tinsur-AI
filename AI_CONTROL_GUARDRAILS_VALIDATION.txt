AI Control Guardrails Validation
================================

Scope
-----
Implemented production-grade AI action controls so insurance AI agents and the general chatbot can assist with recommendations, explanations, triage, and document drafting while deterministic service boundaries remain responsible for binding policies, cancelling policies, taking or waiving payments, creating legal claim records, settling claims, and changing legal policy records.

Design Summary
--------------
- Added a centralized AI action-control service that classifies restricted consequential insurance operations and returns stable deterministic-handoff decisions.
- Restricted policy-agent binding and cancellation tools now return structured handoff responses instead of mutating policies.
- Restricted support-agent cancellation, automated claim-registration, and late-fee waiver tools now return structured handoff responses instead of changing policy, claim, payment, or schedule records.
- Claims-agent claim creation and settlement creation behavior was refactored into triage and draft/handoff behavior, with legacy mutation code removed rather than merely bypassed.
- Policy, support, and generic AI chat prompts now include the same centralized action-control policy instruction so model-facing behavior aligns with tool-level enforcement.

Test Coverage Added
-------------------
- Restricted operations are denied and require deterministic handoff.
- Assistive/read-only operations remain allowed.
- Restricted handoff responses are stable, JSON-serializable, and auditable.
- Policy, support, and generic chatbot prompt surfaces reference the centralized action-control instruction.
- Restricted agent tools no longer contain active direct mutation patterns for policy binding, cancellation, fee waiver, automated claim creation, or claim settlement creation.

Remaining Risks
---------------
- The implementation enforces deterministic handoff at the currently inspected AI agent and generic chatbot surfaces. Future AI tools that perform consequential actions should import and use the centralized action-control service before exposing those tools to agents.
- The handoff responses identify deterministic target services and next steps, but end-to-end deterministic approval workflows should continue to be validated separately at the service/API layer.
- The focused tests intentionally avoid broad incompatible database fixtures and therefore validate guardrail behavior and source-level mutation removal rather than full multi-agent runtime execution with external AI SDKs.
